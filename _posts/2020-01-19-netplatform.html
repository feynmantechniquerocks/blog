In the first post of this blog. I mentioned that I’m going to learn C# in 2021.


I got myself a book, I decided which Youtube tutorials should I take, I downloaded the Visual Studio and started to study C#.


At the very first minute, I faced with a new term “.NET Platform”. Actually, this was not my first time hearing about “.NET Platform” but I didn’t know what this term was. I assume many learners and maybe some C# developers still don’t know what it is. This is why I wanted to write about it before diving in the syntax, data types, OOP etc.


.NET platform literally is a platform that consists of many different components and it is the platform that our C# applications run on. It's free, open source and cross-platform. It helps developers to build all kind of applications. .NET Platform can be thought of as three layers.

  • Common Infrastructure
  • .NET Standart Library
  • App Models


1. Common Infrastructure

This layer consists of programming languages and compilers, Common language Runtime (CLR), Common intermediate language (CIL) and other tools that are necessary to run your programs.


I mentioned about the programming languages. That’s right - plural. You can use not only C# but also F# and Visual Basic to build applications that run on .NET Platform.


Here is the code execution process that will make things easier to understand:

  1. The program will be converted to Common Intermediate Language by the compiler. .Net framework has compilers for all the supported programming languages. These compilers will help us to translate the program to CIL.
  2. To execute the compiled code on a real computer, Common Language Runtime (CLR) which has JIT(just-in-time) compiler will translate the CIL to native code.
  3. A real computer will run this native code.

Thanks to .NET Platform, we don’t need to worry about the computers, operating system etc.


2. .NET Standart Library

Also known as Base Class Library. This library has vast amount of code that can be used in NET Applications.


Many programs requires primitives, collections, file system interactions, IO, Networking, threading and much more. .NET Standary Library provide us many tools to deal with them thus shorten the development time.


3. App Models

App model is a library for creating specific applications. App model was built on top of .Net Standart Library. They can be thought of as high level libraries that focus on solving specific problems for a specific application development. Here are some examples:

  • Web App model> ASP.NET
  • GUI App model > WPF or Windows Forms, UWP
  • Mobile App model > Xamarin
  • Game Development model > Unity

Once you learn one of the languages of .NET Platform, you can specifically focus on the app model library that you’d need. For example, If you’re going to work on the web, there is no point learning Xamarin.


If you check the main image at the top, you’d see three stacks of .NET Platform. I think it’d be nice to mention about them too.

  • The .NET Framework is the oldest and most popular. It runs on Windows devices.
  • The .NET Core is the newest and it’s cross-platform.
  • The Xamarin aims at mobile devices (iOS and Android).


Conclusion

Basically, .NET Platform is a developer platform which consists programming languages and libraries.
