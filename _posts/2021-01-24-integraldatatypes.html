In this post, I'm going to discuss about integral data types which are byte, short, int, long, sbyte,ushort, uint, ulong and char


We use integral types (sometimes referred as integer types) to store numeric values.


Byte

Let's start with byte.


{% highlight c# %} //Let's create a byte variable. byte myByteVar = 2; //Let's see the maximum and minimum value a byte can store. Console.WriteLine("Max. Value: {0}", byte.MaxValue); Console.WriteLine("Min. Value: {0}", byte.MinValue); {% endhighlight %}

And output will be:

{% highlight c %} Max. Value: 255 Min. Value: 0 {% endhighlight %}

Byte type stores 8-bit unsigned(I'll get to that later) int from 0 to 255.


Short

Let's continue with short.

{% highlight c# %} //Let's create a short variable. short myShortVar = 23; //Let's see the maximum and minimum value a short can store. Console.WriteLine("Max. Value: {0}", short.MaxValue); Console.WriteLine("Min. Value: {0}", short.MinValue); {% endhighlight %}

And output will be:

{% highlight c %} Max. Value: 32767 Min. Value: -32768 {% endhighlight %}

Short type stores 16-bit signed int from -32768 to 32767. If you are coming from hardware background. You'll see this is a WORD.


Integer

Let's continue with integer. Int data type could be the most used one. I'll get to why later.

{% highlight c# %} //Let's create a integer variable. int myIntVar = 58; //Let's see the maximum and minimum value an integer can store. Console.WriteLine("Max. Value: {0}", int.MaxValue); Console.WriteLine("Min. Value: {0}", int.MinValue); {% endhighlight %}

And output will be:

{% highlight c %} Max. Value: 2147483647 Min. Value: -2147483648 {% endhighlight %}

Integer type stores 32-bit signed int from -2147483648 to 2147483647. If you are coming from hardware background. You'll see this is a double word (DWORD).


Long

If you have so much to store, long type is the way to go.

{% highlight c# %} //Let's create a long variable. long myLongVar = 58; //Let's see the maximum and minimum value an long can store. Console.WriteLine("Max. Value: {0}", long.MaxValue); Console.WriteLine("Min. Value: {0}", long.MinValue); {% endhighlight %}

And output will be:

{% highlight c %} Max. Value: 9223372036854775807 Min. Value: -9223372036854775808 {% endhighlight %}

Long type stores 32-bit signed int from -9223372036854775808 to 9223372036854775807. This is huge.


Signed/Unsigned Types

If a type is signed, it means it can be positive or negative. If a type is unsigned, it means it can be only positive. This would allow us to go twice as high but of course there is a downside, we can't use negative values.


We've covered short, int, and long types. If you check their ranges you'll see that they can be positive and negative. That means they are signed types. Byte on the other hand can store value between 0-255. That means it is unsigned type.


All these types also have unsigned counterparts. (Signed counterpart for byte). Let's learn about them as well.


sByte

This is signed version of the byte type.

{% highlight c# %} //Let's create a sbyte variable. sbyte mysByteVar = -10; //Let's see the maximum and minimum value an sbyte can store. Console.WriteLine("Max. Value: {0}", sbyte.MaxValue); Console.WriteLine("Min. Value: {0}", sbyte.MinValue); {% endhighlight %}

And output will be:

{% highlight c %} Max. Value: 127 Min. Value: -128 {% endhighlight %}

sByte type stores 8-bit signed int from -128 to 127.


uShort

This is unsigned version of the short type.

{% highlight c# %} //Let's create a ushort variable. ushort mysByteVar = 280; //Let's see the maximum and minimum value a ushort can store. Console.WriteLine("Max. Value: {0}", ushort.MaxValue); Console.WriteLine("Min. Value: {0}", ushort.MinValue) {% endhighlight %}

And output will be:

{% highlight c %} Max. Value: 65535 Min. Value: 0 {% endhighlight %}

sByte type stores 16-bit unsigned int from 0 to 65535.


uInteger

This is unsigned version of the integer type.

{% highlight c# %} //Let's create a uint variable. uint myuIntVar = 280; //Let's see the maximum and minimum value a uint can store. Console.WriteLine("Max. Value: {0}", uint.MaxValue); Console.WriteLine("Min. Value: {0}", uint.MinValue); {% endhighlight %}

And output will be:

{% highlight c %} Max. Value: 4294967295 Min. Value: 0 {% endhighlight %}

uInt type stores 32-bit unsigned int from 0 to 4294967295.


uLong

This is unsigned version of the long type.

{% highlight c# %} //Let's create a ulong variable. ulong myuLongVar = 280; //Let's see the maximum and minimum value a ulong can store. Console.WriteLine("Max. Value: {0}", ulong.MaxValue); Console.WriteLine("Min. Value: {0}", ulong.MinValue); {% endhighlight %}

And output will be:

{% highlight c %} Max. Value: 18446744073709551615 Min. Value: 0 {% endhighlight %}

uLong type stores 64-bit unsigned int from 0 to 18446744073709551615.


Char

We are almost done with integral types. We only have one more to discuss about. It's Char type. It's used to store single character or a letter. You might think why we cover "char" as integral type since it's used to store characters.


It's because, under the hood, each char is assigned to a single number. When we use char, the computer will know we use it as a letter so it will treat char as letter not number. For example if you print char, the computer will print out the letter not the number.

{% highlight c# %} //Let's create a char variable. char myCharVar = 'c'; char anotherCharVar = '&'; //Let's print the char variables on the console. Console.WriteLine("myCharVar: {0}" , myCharVar); Console.WriteLine("anotherCharVar: {0}", anotherCharVar); {% endhighlight %}

And output will be:

{% highlight c %} myCharVar: c myCharVar: & {% endhighlight %}

char type stores 16-bit data.
