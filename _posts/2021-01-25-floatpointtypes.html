In the math world, there are real numbers. Counterparts of real numbers in C# are float, double, and decimal.


If we didn't have real numbers, we wouldn't be able to store data such as 2.8 .


Double

Double stores 8 byte of data. It has 15 or 16 digits of precision. Double is the most precise data type.


{% highlight c# %} //Let's create a float variable. double myDoubleVariable = 12.34546874; //Let's see the maximum and minimum value a double can store. Console.WriteLine("myDoubleVariable: {0}", myDoubleVariable); Console.WriteLine("Max. Value: {0}" , double.MaxValue); Console.WriteLine("Min Value: {0}", double.MinValue); {% endhighlight %}

And output will be:

{% highlight c %} myDoubleVariable: 12.34546874 Max. Value: 1.79769313486232E+308 Min Value: -1.79769313486232E+308 {% endhighlight %}

Float

Float stores 4 byte of data. It has 7 digits of precision.


{% highlight c# %} //Let's create a float variable. float myFloatVariable = 12.34546f; //Let's see the maximum and minimum value a float can store. Console.WriteLine("myFloatVariable: {0}", myFloatVariable); Console.WriteLine("Max. Value: {0}" , float.MaxValue); Console.WriteLine("Min Value: {0}", float.MinValue); {% endhighlight %}

And output will be:

{% highlight c %} myFloatVariable: 12.34546 Max. Value: 3.402823E+38 Min Value: -3.402823E+38 {% endhighlight %}

Have you seen the f at the end of variable definition ? When you define a float point variable, you have to put f at the end of the definition.


If you don't put the f at the end, the compiler will treat it as double. We specifically have to tell the compiler it is float not double.


Decimal

Last data type for floating point types is decimal. This data type was built to make money related calculations but it doesn't have any hardware-level support so it's slow but it doesn't have losing accuracy issue that float and double have.


It can store 16 bytes of data and has 28 or 29 digits of precision.

{% highlight c# %} //Let's create a decimal variable. decimal myDecimalVariable = 12.345m; //Let's see the maximum and minimum value a decimal can store. Console.WriteLine("myDecimalVariable: {0}", myDecimalVariable); Console.WriteLine("Max. Value: {0}" , decimal.MaxValue); Console.WriteLine("Min Value: {0}", decimal.MinValue); {% endhighlight %}

And output will be:

{% highlight c %} myDecimalVariable: 12.345 Max. Value: 79228162514264337593543950335 Min Value: -79228162514264337593543950335 {% endhighlight %}

Just like float, we need to put m at the end of variable definition when we define a decimal type variable.


If you don't put the f at the end, the compiler will treat it as double. We specifically have to tell the compiler it is decimal not double.


Some people say that decimal is not a floating point type but I think it's fine to mention about it under this title.
