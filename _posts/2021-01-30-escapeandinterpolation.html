This is going to be a short post about escape character and string interpolation.


As you know we've been writing console applications so far. It's very common to print something on the console or ask for user input. I believe both escape character and string interpolation will be very useful for our console programs so I think I have to address them at this point.


Escape Character

If you're working with strings, you will have to use escape character eventually. If we didn't have escape characters in programming languages, we wouldn't be able to print some special characters such as quotes(").


Let's see the escape character in action. Let our program to print out quotes to console.


{% highlight c# %} // Print " to the console. Console.WriteLine(" \" "); // Print "Hello World!" to the console. Console.WriteLine(" \"Hello World! \" "); {% endhighlight %}

And output will be:

{% highlight c %} " "Hello World! " {% endhighlight %}

As you can see above, backslash is the escape character for C#. If you start the string sequence by using it, the compiler will treat the next character as plain text.


We have an edge case here. Imagine that you need to print backslash. Maybe you need to show the user a path on the system. What's going to happen then ? Well, in that case, just use double backslash.


{% highlight c# %} // Print \ on the console. Console.WriteLine(" C:\\Users\\Documents "); {% endhighlight %}

And output will be:

{% highlight c %} C:\Users\Documents {% endhighlight %}

String Interpolation

I've used two techniques to display something to users. One of them is string format.


{% highlight c# %} int myVar = 12; Console.WriteLine("My variable is: {0}", myVar); {% endhighlight %}

The other one is string concatenation.


{% highlight c# %} int myVar = 12; Console.WriteLine("My variable is: " + myVar); {% endhighlight %}

These methods are fine and there is nothing wrong using them. However they are not my favourite methods. I like the string interpolation most. C# 6.0 introduced this concept.


Here is an example of string interpolation.


{% highlight c# %} int myVar = 12; Console.WriteLine($"My variable is: {myVar}"); {% endhighlight %}

We put a dollar sign($) before double quotes and that tells the compiler this isn't a regular string. It has some code that needs to be evaulated.
