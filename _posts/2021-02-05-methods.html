Programming is all about solving problems. In real life, one of the best way to solve a big problem is to break it down into smaller pieces. This method also applies programming problems. C# provides us methods to do so. Methods are also known as functions, subroutines and procedures in other programming languages.


Methods are one of the core parts of C#. In fact, we have been using them since the beginning. We always print some text on the console, right ? WriteLine is the name of the method and it belongs to Console class. We will talk about classes in the next posts.


Creating and Calling Methods

Let's create a method.


{% highlight c# %} static void sayHello() { Console.WriteLine("Greetings !"); } {% endhighlight %}

That's it. We've just created our first method. It just prints Greetings ! on the console. There are a few things to address at the moment.

  • 1. Static: Let's skip this for now.
  • 2. Void: That means our method will not return anything. I will show you how to return something in a minute.
  • 3. Method Name: We have to name our method so we can call them later. Our method name is sayHello.
  • 4. Paranthesis: Parenthesis are for arguments. sayHello doesn't require any arguments.

We have our sayHello method but we haven't called it yet. In order to make our method work, we need to call it in our main program.

{% highlight c# %} using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Methods { class Program { static void Main(string[] args) { sayHello(); } static void sayHello() { Console.WriteLine("Greetings !"); } } } {% endhighlight %}

Return Something from a Method

We usually need methods to make some sort of calculations for us so we need methods to return something.


Here is an example:


{% highlight c# %} namespace Method { class Program { static void Main(string[] args) { int myRandomNumber = randomIntGenerator(); Console.WriteLine($"My random number is: {myRandomNumber}."); } static int randomIntGenerator() { return 71; } } } {% endhighlight %}

Here you can see, we changed void with int. That tells the compiler our method will return an integer. We can assign the return value to a variable and then use it.


Method Arguments

Methods are supposed to be reusable so we need them to be used over and over again within different situations. That's why we need to give them arguments and return a value depends on the arguments provided.


We had a small program in the past to calculate the area of the given triangle. Let's write a method that calculates the area of a triangle. We will need the base and the height as the arguments.

{% highlight c# %} class Program { static void Main(string[] args) { float area = areaCalculator(17.2f, 5.6f); Console.WriteLine($"Area of the given triangle is: {area}."); Console.ReadLine(); } static float areaCalculator(float baseValue, float height) { float area = (baseValue * height) / 2; return area; } } {% endhighlight %}

The output will be:


{% highlight c %} Area of the given triangle is: 48.16. {% endhighlight %}
Methods that have different number of arguments can have the same name. This can be tricky !
